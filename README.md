# Spotify User Behavior Analysis

### 🎯 Project Objective
The goal of this study is to identify behavioral "signals" that differentiate 
Premium and Free users. By analyzing song-skip rates and session lengths across 
different devices, this project aims to provide actionable insights for 
product strategy.

### 🔍 Research Philosophy
I am driven by the belief that data is a human story told in aggregate. My 
analytical foundation taught me how to find signals in the noise, but my work 
in UX taught me that those signals represent real people. My goal is to ensure 
that the voice of the user is never lost in the math.

### 🛠️ Technical Stack
- Language: R (Tidyverse) for data cleaning and statistical analysis.
- Logic: SQL for data architecture and conceptual query design.
- Reporting: Quarto / RMarkdown for reproducible research documentation.

### 📈 Project Status
Active Development

I am currently in the Data Ingestion and Pipeline Mapping phase. While this specific project utilizes a flat-file dataset from Kaggle, I am documenting the SQL logic required to extract similar telemetry from a relational production environment. My current focus is establishing a reproducible R-based workflow to identify significant behavioral trends.
